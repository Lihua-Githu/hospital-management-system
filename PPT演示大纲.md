## 幻灯片14：SQL查询示例
### 标题：关键SQL - 收入统计
**按科室统计收入**：
```sql
SELECT d.dept_name,
       COUNT(DISTINCT v.visit_id) as visit_count,
       SUM(b.total_fee) as total_revenue
FROM billing b
JOIN visit v ON b.visit_id = v.visit_id
JOIN department d ON v.dept_id = d.dept_id
WHERE b.payment_status = '已支付'
GROUP BY d.dept_id, d.dept_name
ORDER BY total_revenue DESC;
```

**特点**：
- 多表关联查询
- 聚合函数应用
- 结果排序优化

## 幻灯片15：完整性约束实现
### 标题：数据完整性保障
**四类约束**：

**实体完整性**
```sql
patient_id INT PRIMARY KEY AUTO_INCREMENT
```

**参照完整性**
```sql
FOREIGN KEY (dept_id) REFERENCES department(dept_id)
```

**用户定义完整性**
```sql
status ENUM('等待就诊', '就诊中', '已完成', '已离院')
```

**业务完整性**
- 费用金额 >= 0
- 预约时间 >= 当前时间
- 排班时间不冲突

### 标题：开发中的挑战与解决
**三个难点**：

**难点1：并发控制**
- 问题：多人同时预约可能冲突
- 解决：唯一索引 + 事务处理

**难点2：数据一致性**
- 问题：结算时需同步更新多张表
- 解决：使用数据库事务保证原子性

**难点3：查询性能**
- 问题：统计查询涉及多表联合
- 解决：合理设置索引，优化SQL语句
